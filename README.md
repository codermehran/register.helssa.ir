# register.helssa.ir

سامانه ثبت‌نام اولیه بیماران برای طرح پزشک خانواده دکتر حسین شبانی. پروژه با Django ساخته شده و یک صفحه فارسی و راست‌به‌چپ برای دریافت اطلاعات اولیه بیمار، ثبت داده‌ها، ارسال پیامک کاوه‌نگار و گزارش‌گیری مدیریتی فراهم می‌کند.

## امکانات

- فرم ثبت‌نام فارسی و راست‌به‌چپ برای نام، نام خانوادگی، کد ملی و شماره موبایل
- اعتبارسنجی شماره موبایل ایران:
  - الزام به ۱۱ رقم
  - فقط عدد
  - شروع با `09`
  - جلوگیری از ثبت شماره تکراری
- اعتبارسنجی کد ملی:
  - الزام به ۱۰ رقم
  - فقط عدد
  - جلوگیری از ثبت کد ملی تکراری
- نمایش خطاهای هر فیلد و پیام موفقیت به زبان فارسی
- جلوگیری از ارسال چندباره فرم با غیرفعال کردن دکمه ثبت پس از ارسال
- متادیتای اشتراک‌گذاری، تصویر معرفی و لوگوی سایت برای پیش‌نمایش لینک
- شمارنده نمایشی ثبت‌نام‌ها در صفحه عمومی
- پنل مدیریت Django برای جست‌وجو و مدیریت بیماران
- ارسال پیامک ثبت‌نام پس از ایجاد بیمار با کاوه‌نگار
- اکشن ادمین برای ارسال پیامک «انجام شد» به بیماران انتخاب‌شده
- لاگ پیامک‌ها همراه با وضعیت موفق یا ناموفق و پاسخ سرویس
- خروجی PDF از بیماران انتخاب‌شده در پنل ادمین
- رهگیری رویدادهای بازدید و ثبت‌نام بدون ذخیره مستقیم داده‌های حساس فرم در فراداده
- گزارش بازدید سایت در ادمین با بازه زمانی، خلاصه آماری و خروجی PDF
- تست‌های واحد برای مدل، فرم، نما، پیامک، ادمین، PDF، قالب و گزارش بازدید

## فناوری‌ها

- Python
- Django `5.2.14`
- SQLite برای توسعه محلی
- MySQL اختیاری برای محیط اجرا
- Kavenegar برای پیامک
- ReportLab برای ساخت PDF
- `arabic-reshaper` و `python-bidi` برای خروجی PDF فارسی
- HTML/CSS/JavaScript ساده برای رابط کاربری عمومی

## پیش‌نیازها

## نصب و راه‌اندازی محلی

وارد پوشه پروژه شوید:

```powershell
cd C:\Users\Bartar\OneDrive\Apps\Desktop\register.helssa.ir
```

محیط مجازی بسازید و فعال کنید:

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
```

وابستگی‌ها را نصب کنید:

```powershell
pip install -r requirements.txt
```

فایل محیطی نمونه را کپی کنید:

```powershell
Copy-Item .env.example .env
```

مایگریشن‌ها را اجرا کنید:

```powershell
python manage.py migrate
```

سرور توسعه را اجرا کنید:

```powershell
python manage.py runserver
```

سپس پروژه در این آدرس در دسترس است:

```text
http://127.0.0.1:8000/
```

## مدل‌های داده

### Patient

| فیلد | نوع | توضیح |
| --- | --- | --- |
| `first_name` | `CharField(max_length=100)` | نام بیمار |
| `last_name` | `CharField(max_length=100)` | نام خانوادگی بیمار |
| `mobile` | `CharField(max_length=11, unique=True)` | شماره موبایل یکتا |
| `national_code` | `CharField(max_length=10, unique=True, null=True, blank=True)` | کد ملی یکتا |
| `created_at` | `DateTimeField(auto_now_add=True)` | زمان ثبت رکورد |

### SMSMessageLog

لاگ ارسال پیامک‌ها را نگه می‌دارد؛ شامل بیمار، شماره موبایل، قالب پیامک، توکن، وضعیت، پاسخ سرویس، خطا و زمان ارسال.

### VisitEvent

رویدادهای بازدید و ثبت‌نام را برای گزارش مدیریتی ذخیره می‌کند؛ شامل شناسه بازدیدکننده، نوع رویداد، مسیر، مرورگر، ارجاع‌دهنده، هش آی‌پی، وضعیت پاسخ، بیمار مرتبط و فراداده امن.

## تنظیمات محیطی

تنظیمات از فایل `.env` خوانده می‌شوند و اگر متغیری وجود نداشته باشد، مقدار پیش‌فرض توسعه محلی در `config/settings.py` استفاده می‌شود.

متغیرهای اصلی:

```env
DJANGO_SECRET_KEY=change-me
DJANGO_DEBUG=True
DJANGO_ALLOWED_HOSTS=localhost,127.0.0.1
DJANGO_CSRF_TRUSTED_ORIGINS=
DJANGO_LANGUAGE_CODE=fa-ir
DJANGO_TIME_ZONE=Asia/Tehran
DJANGO_STATIC_URL=static/
DJANGO_STATIC_ROOT=statics
```

### پایگاه داده

پروژه به‌صورت پیش‌فرض از SQLite استفاده می‌کند:

```env
DB_ENGINE=sqlite
SQLITE_NAME=db.sqlite3
```

برای MySQL:

```env
DB_ENGINE=mysql
MYSQL_DATABASE=register_helssa
MYSQL_USER=root
MYSQL_PASSWORD=your-password
MYSQL_HOST=127.0.0.1
MYSQL_PORT=3306
MYSQL_CHARSET=utf8mb4
```

## پیامک کاوه‌نگار

بعد از ثبت موفق بیمار، سیگنال `post_save` پیامک قالبی ثبت‌نام را ارسال می‌کند. در پنل ادمین نیز می‌توان بیماران را انتخاب کرد و اکشن ارسال پیامک «انجام شد» را اجرا کرد.

تنظیمات لازم:

```env
KAVENEGAR_API_KEY=your-api-key
KAVENEGAR_REGISTER_TEMPLATE=register
KAVENEGAR_DONE_TEMPLATE=register-done
```

توکن پیامک از نام و نام خانوادگی بیمار ساخته می‌شود و فاصله‌ها با `_` جایگزین می‌شوند. اگر ارسال پیامک خطا بدهد، ثبت‌نام یا اکشن ادمین متوقف نمی‌شود و نتیجه در `SMSMessageLog` ثبت می‌شود.

## گزارش‌ها و PDF

در پنل ادمین:

- از فهرست بیماران می‌توان گزارش PDF بیماران انتخاب‌شده را دانلود کرد.
- از بخش «گزارش بازدید سایت» می‌توان بازه زمانی انتخاب کرد، خلاصه آماری دید و خروجی PDF جزئیات را گرفت.

برای تولید PDF فارسی، پروژه به فونت‌های قابل استفاده توسط ReportLab نیاز دارد. پروژه ابتدا مسیرهای تنظیم‌شده در Django settings را بررسی می‌کند و سپس چند مسیر رایج پروژه‌ای و سیستمی مثل DejaVu، Tahoma و Arial را امتحان می‌کند. در صورت نیاز می‌توان تنظیمات زیر را در Django settings اضافه کرد:

```python
PDF_FONT_PATH = "path/to/DejaVuSans.ttf"
PDF_FONT_BOLD_PATH = "path/to/DejaVuSans-Bold.ttf"
```

## نکات توسعه

- فایل استایل اصلی صفحه عمومی در `patients/static/patients/css/style.css` است.
- قالب فرم ثبت‌نام در `patients/templates/patients/register.html` است.
- راهنمای تصاویر لوگو و اشتراک‌گذاری در `logo.md` است.
- منطق ثبت اطلاعات در `patients/views.py` و اعتبارسنجی‌ها در `patients/forms.py` پیاده‌سازی شده‌اند.
- تغییرات مربوط به پیامک معمولاً در `patients/sms.py`، `patients/signals.py`، `patients/sms_logs.py` و `patients/admin.py` پخش می‌شوند.
- تغییرات مربوط به گزارش بازدید معمولاً در `patients/analytics.py`، `patients/middleware.py` و `patients/admin.py` هستند.
- پس از تغییر مدل‌ها، مایگریشن بسازید و اجرا کنید:

```powershell
python manage.py makemigrations
python manage.py migrate
```

## آماده‌سازی برای انتشار

قبل از استفاده عملیاتی:

- مقدار `DJANGO_SECRET_KEY` را امن و اختصاصی تنظیم کنید.
- `DJANGO_DEBUG` را `False` کنید.
- `DJANGO_ALLOWED_HOSTS` و `DJANGO_CSRF_TRUSTED_ORIGINS` را برای دامنه واقعی تنظیم کنید.
- پایگاه داده مناسب تولید و بکاپ منظم را آماده کنید.
- مسیر و سرویس‌دهی فایل‌های استاتیک را تنظیم کنید.
- HTTPS، لاگ‌گیری، مانیتورینگ و محدودسازی دسترسی ادمین را کامل کنید.
- درباره نگهداری داده‌های بیماران، سطح دسترسی کاربران و سیاست حذف یا بایگانی داده‌ها تصمیم مشخص بگیرید.
- برای رهگیری بازدید، در صورت نیاز `ANALYTICS_IP_HASH_SALT` اختصاصی تنظیم کنید.

## خطاهای رایج

### Django نصب نیست

محیط مجازی را فعال و وابستگی‌ها را نصب کنید:

```powershell
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
```

### جدول‌های پایگاه داده وجود ندارند

مایگریشن‌ها را اجرا کنید:

```powershell
python manage.py migrate
```

### ثبت شماره موبایل یا کد ملی انجام نمی‌شود

شماره موبایل باید دقیقاً ۱۱ رقم، فقط عدد و با `09` شروع شود. کد ملی باید دقیقاً ۱۰ رقم و فقط عدد باشد. هر شماره موبایل و هر کد ملی فقط یک‌بار قابل ثبت است.

### پیامک ارسال نمی‌شود

مطمئن شوید `KAVENEGAR_API_KEY` و نام قالب‌ها تنظیم شده‌اند. خطاهای ارسال در لاگ برنامه و مدل `SMSMessageLog` قابل بررسی هستند.

### خروجی PDF خطای فونت می‌دهد

فونت‌های مناسب فارسی مثل DejaVu، Tahoma یا Arial را نصب کنید یا مسیر فونت‌ها را با `PDF_FONT_PATH` و `PDF_FONT_BOLD_PATH` در تنظیمات Django مشخص کنید.
