# AGENTS.md

راهنمای نگهداری و توسعه پروژه `register.helssa.ir` برای عامل‌های کدنویسی.

## خلاصه پروژه

این پروژه یک اپ Django برای ثبت‌نام اولیه بیماران طرح پزشک خانواده است. رابط عمومی فارسی و راست‌به‌چپ است و اطلاعات نام، نام خانوادگی، کد ملی و شماره موبایل را دریافت می‌کند. پنل ادمین برای مدیریت بیماران، ارسال پیامک، مشاهده لاگ پیامک، خروجی PDF و گزارش بازدید استفاده می‌شود.

## دستورهای رایج

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
python manage.py migrate
python manage.py runserver
python manage.py test
```

پس از تغییر مدل‌ها:

```powershell
python manage.py makemigrations
python manage.py migrate
python manage.py test
```

## ساختار مهم

- `config/settings.py`: تنظیمات Django، خواندن `.env`، دیتابیس، استاتیک، پیامک و رهگیری بازدید
- `config/urls.py`: مسیرهای اصلی پروژه
- `patients/models.py`: مدل‌های `Patient`، `SMSMessageLog`، `VisitEvent` و پروکسی `VisitReport`
- `patients/forms.py`: فرم ثبت‌نام و اعتبارسنجی کد ملی و موبایل
- `patients/views.py`: نمایش و پردازش فرم ثبت‌نام، متادیتای اشتراک‌گذاری و آمار صفحه
- `patients/admin.py`: پنل ادمین، اکشن‌های پیامک، خروجی PDF و گزارش بازدید
- `patients/sms.py`: ساخت توکن و ارسال پیامک کاوه‌نگار
- `patients/signals.py`: ارسال پیامک ثبت‌نام پس از commit ایجاد بیمار
- `patients/sms_logs.py`: ثبت نتیجه ارسال پیامک
- `patients/analytics.py`: ثبت و خلاصه‌سازی رویدادهای بازدید
- `patients/middleware.py`: رهگیری GETهای عمومی
- `patients/templates/patients/register.html`: صفحه عمومی ثبت‌نام
- `patients/static/patients/css/style.css`: استایل صفحه عمومی
- `patients/tests.py`: تست‌های واحد پروژه
- `logo.md`: مشخصات تصاویر لوگو و پیش‌نمایش اشتراک‌گذاری

## قواعد توسعه

- رابط کاربری عمومی باید فارسی، راست‌به‌چپ و قابل استفاده در موبایل بماند.
- پیام‌های خطا و موفقیت فرم باید فارسی و روشن باشند.
- فیلدهای حساس مثل شماره موبایل، کد ملی، نام و نام خانوادگی را در فراداده رهگیری بازدید ذخیره نکنید.
- رفتار ارسال پیامک نباید ثبت‌نام بیمار یا اکشن‌های ادمین را متوقف کند؛ خطا باید لاگ شود.
- برای ارسال پیامک فقط از تنظیمات `KAVENEGAR_API_KEY`، `KAVENEGAR_REGISTER_TEMPLATE` و `KAVENEGAR_DONE_TEMPLATE` استفاده کنید.
- توکن کاوه‌نگار از نام و نام خانوادگی ساخته می‌شود و فاصله‌ها با `_` جایگزین می‌شوند.
- تغییرات مدل باید همراه مایگریشن باشند.
- برای تغییرات فرم، ادمین، پیامک، گزارش یا PDF تست مرتبط اضافه یا به‌روزرسانی کنید.
- از ذخیره کلید API، رمز، دیتابیس واقعی یا داده بیمار در مخزن خودداری کنید.
- در پایان تسک خود حتما یک کامیت پیشنهادی ارائه دهید.

## نکات تست

قبل از تحویل تغییرات، حداقل این دستور را اجرا کنید:

```powershell
python manage.py test
```

تست‌های PDF در محیط تست فونت‌های ReportLab را mock/register می‌کنند. در محیط واقعی اگر خروجی PDF خطای فونت داد، مسیر فونت‌ها را در settings با `PDF_FONT_PATH` و `PDF_FONT_BOLD_PATH` تنظیم کنید.

## دیتابیس و محیط

پروژه به‌صورت پیش‌فرض SQLite استفاده می‌کند. برای MySQL، متغیر `DB_ENGINE=mysql` و متغیرهای `MYSQL_*` را تنظیم کنید. تنظیمات محلی از `.env` خوانده می‌شوند و نمونه آن در `.env.example` قرار دارد.

## رهگیری بازدید

رهگیری بازدید با `PublicVisitTrackingMiddleware` و مدل `VisitEvent` انجام می‌شود. شناسه بازدیدکننده در کوکی `helssa_vid` ذخیره می‌شود. آی‌پی به‌صورت هش ذخیره می‌شود مگر اینکه صراحتاً `ANALYTICS_STORE_RAW_IP` فعال شده باشد. هنگام افزودن رویداد جدید، داده خام فرم را در `metadata` قرار ندهید.

## گزارش‌های ادمین

- اکشن `download_patients_pdf_report` از بیماران انتخاب‌شده PDF می‌سازد.
- اکشن `send_done_sms_to_patients` پیامک قالب انجام شد را برای بیماران انتخاب‌شده می‌فرستد.
- مدل پروکسی `VisitReport` صفحه گزارش بازدید سایت را در ادمین فراهم می‌کند.
- گزارش PDF بازدید با `VISIT_REPORT_PDF_MAX_EVENTS` محدود می‌شود.
